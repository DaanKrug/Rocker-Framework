JSON data format - Loading a single object to put in a RockerForm

JSON format for read a object to a RockerForm (operation 'read'):
{"keyId(1)":"keyValue(1)", ..., "keyId(n)":"keyValue(n)"}

Where { } delimits the object scope, and the "" : "" pairs delimit the object name/value attributes.

Example a object 'Person':
- id = 10;
- name = 'John';
- age = 40;

turn into:
{"id":"10","name":"John","age":"40"}

JSON data format - Loading a list objects to put into a RockerTable

JSON format for load a object list to a RockerTable (operation 'filterAndLoad'):
{"totalObjects": totalObjectsNumber, "objects": {1}{2}{3}...{n}}

where:
totalObjects is a C.O.C key identifier that represents a count of objects in a database (as 'select count(*) from ' SQL ),
totalObjectsNumber is a number of objects into a respective table on database,
objects is a C.O.C key identifier that represent the paginated list objects returned on request, and {1}{2}{3}...{n} is a list of json objects following a Loading a single object to put in a RockerForm sintax

C.O.C (Convention over Configuration) terms - totalObjects

C.O.C (Convention over Configuration) terms - objects

C.O.C (Convention over Configuration) terms - operation

operation:
describes a operation name on request. Used in RockerForm and RockerTable.
- RockerForm use could be explored in file rockstars.js
arrayFields[i] = new Hidden('operation','store');
- RockerTable has a internal mechanism to use it, whit 'filterAndLoad', 'read' and 'delete' operations for contained objects.

C.O.C (Convention over Configuration) terms - id

id:
describes a Object Id. Used in RockerForm and RockerTable.
- RockerForm use could be explored in file rockstars.js

- RockerTable has a internal mechanism to use it, whit 'read' and 'delete' operations for contained objects.

C.O.C (Convention over Configuration) terms - additionalParamsAndValuesOnRequestTable

additionalParamsAndValuesOnRequestTable:
Attribute of RockerTable, that allow change the filtering and loading data mechanism.
- Need be used whit &operation=filterAndLoad.

C.O.C (Convention over Configuration) terms - filterAndLoad

filterAndLoad:
its a 'operation' used to load data by a RockerTable (url request to load data always contains '&operation=filterAndLoad').
- Used internally by a RockerTable to default filter and load data operations.
    - Use the following conventioned terms internally:
       - numberFilterParams: parameter that determine the number of columns that are used to filter process.        - attr0, attr1 ... attrN: parameters that determine the name of attributes used to filter in columns in (0,1 ... N).
       - valueAttr0, valueAttr1 ... valueAttrN: parameters that determine the values respectives for each of attributes names of attr0, attr1 ... attrN.
      Parameter valueAttr0 contains the value associated the attr0 parameter and follow like this until the last parameter(valueAttrN).

This filtering should be prepared in back-end server, as could be explored in file node.js.appserver.js :
...
var numberFilterParams = dataQuery.numberFilterParams;
var filterParams = new Array();
for(var i = 0; i < numberFilterParams; i++){// dinamic filter according input fields on table header
filterParams[filterParams.length] = [dataQuery[ 'attr' + i],dataQuery[ 'valueAttr' + i],null];
}
...

- To personalize the way of load and filter data, for one specific situation or for change it for all cases for a RockerTable, should be used to set it, the 'additionalParamsAndValuesOnRequestTable' RockerTable attribute. It could be explored in file rockstars.js
...
this.additionalParamsAndValuesOnRequestTable = '';
this.additionalParamsAndValuesOnRequestTable += '&operation=filterAndLoad';
...
